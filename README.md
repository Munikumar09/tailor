# AI Job Application Commander

An end-to-end, AI-powered job application pipeline. It ingests job listings, scores their fit against your master resume using semantic similarity, and runs a five-system LangGraph agent that tailors your resume — preserving all original formatting — and produces a ready-to-send `.docx` file.

## Feature Overview

| Area | What it does |
|------|-------------|
| **Job ingestion** | Fetches listings from JSearch (RapidAPI), deduplicates, and scores each against your resume |
| **Semantic fit scoring** | Cosine similarity of `all-MiniLM-L6-v2` embeddings — no keyword matching, no hard-requirement penalties |
| **Match reason** | Gemini Flash summarises skill-alignment in two sentences (separate from the score) |
| **Resume tailoring** | Five-system LangGraph agent: JD structuring → gap analysis → bullet rewriting → docx export → ATS scoring |
| **Formatting preservation** | Tailored output is generated by mutating the original `.docx` XML, keeping every font, margin, and style intact |
| **Master resume management** | Upload, preview (paper-like rendering with correct fonts, headings, bullets), edit in-browser, version history, export |
| **Keyword gap analysis** | NLP phrase extraction (spaCy) + semantic matching (sentence-transformers) + verb-inflection normalisation |
| **ATS analytics** | Before/after ATS score delta, keyword coverage, pass-band classification |

## Quick Start

### Prerequisites

| Tool | Version |
|------|---------|
| Python | 3.12+ |
| Node.js | 18+ |
| npm | 9+ |

You will need API keys for:
- **Google Gemini** — LLM backbone (JD structuring, bullet rewriting, match reasons)
- **Groq** — LLM fallback (automatic, no code change needed)
- **RapidAPI / JSearch** — Job listing source

### 1 — Backend

```bash
cd backend
python3 -m venv venv
source venv/bin/activate          # Windows: .\venv\Scripts\activate

pip install -r requirements.txt

cp .env.example .env
# Fill in GEMINI_API_KEY, GROQ_API_KEY, RAPIDAPI_KEY in .env

uvicorn main:app --reload         # http://localhost:8000
```

Interactive API docs: [`http://localhost:8000/docs`](http://localhost:8000/docs)

### 2 — Frontend

```bash
cd frontend
npm install
npm run dev                       # http://localhost:3000
```

### 3 — First-time setup (in the UI)

1. Go to **Resume** → upload your master `.docx` resume
2. Go to **Profile** → fill in your details and set `min_ai_score` threshold (default 75)
3. Go to **Ingestion** → configure job search (title, location) and trigger a fetch
4. Jobs above your threshold land in **Jobs** with AI match scores
5. Open a job → click **Tailor** to run the five-system pipeline
6. Download the tailored `.docx` from the job detail page

## Tech Stack

### Backend

| Layer | Technology |
|-------|-----------|
| Framework | FastAPI + Uvicorn |
| Orchestration | LangGraph (stateful agent DAG) |
| LLM | Google Gemini Flash 2.5 (Groq Llama-3.3-70b fallback) |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` |
| NLP | spaCy, rank-bm25 |
| Database | SQLite via SQLModel (SQLAlchemy + Pydantic) |
| Document I/O | python-docx, lxml (direct XML manipulation) |
| Job Search | JSearch API (RapidAPI) |
| ML | scikit-learn, NumPy, PyTorch (CPU) |

### Frontend

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 16 (App Router) |
| Language | TypeScript 5 (strict mode) |
| Styling | Tailwind CSS 4 |
| Icons | Lucide React |
| Server state | TanStack React Query 5 |
| HTTP | Axios |

## Data Flow

```
JSearch API
    │
    ▼
POST /ingest/search
    │  dedup · filter
    ▼
score_job_fit()
    │  all-MiniLM-L6-v2 cosine similarity (score)
    │  Gemini Flash (reason text)
    ▼
Job saved → status: PENDING
    │
    │ (user clicks "Tailor")
    ▼
POST /tailor/{job_id}
    │
    ▼  LangGraph pipeline
    ├─ [1] analyze_gap        → keyword_analysis, extracted_keywords
    ├─ [2] rewrite_bullets    → tailored bullet suggestions (Gemini)
    ├─ [3] validate_changes   → schema check
    ├─ [4] generate_doc       → mutated .docx (XML-level, formatting preserved)
    └─ [5] generate_analytics → ATS score delta, pass-band, coverage report
    │
    ▼
Job updated → status: TAILORED
    │  tailored_resume_path, tailored_bullets, analytics, extracted_keywords
    ▼
Frontend downloads tailored .docx
```

## Environment Variables

All variables live in `backend/.env` (copy from `backend/.env.example`).

| Variable | Required | Description |
|----------|----------|-------------|
| `GEMINI_API_KEY` | Yes | Google Gemini Flash API key |
| `GROQ_API_KEY` | Yes | Groq API key (LLM fallback) |
| `RAPIDAPI_KEY` | Yes | RapidAPI key for JSearch |
| `DATABASE_URL` | No | SQLite path (defaults to `sqlite:///jobs.db`) |

Frontend: set `NEXT_PUBLIC_API_URL` in `frontend/.env.local` if the backend runs on a port other than `8000`.
